Sets don't normally have keys... According to standard array merging rules, later values for the same string keys will overwrite previous ones. This rule is in effect here, but with a set, this can also cause loss of data, depending on the order of the values being merged.

public testMergingSetsOfObjectsWithKeysAlternate2() : void

In this test, $obj2 does not appear in the merged collection because of the order in which the merging occurs. First, $obj2 is found at key "b." When merging the second set, we see $obj2 at key "c," but we find that our set already contains $obj2, so we don't try to add it again. Then, we merge $obj3 to the key "b," which overwrites $obj2.
